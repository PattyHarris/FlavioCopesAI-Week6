Port Pilot

A terminal-first dashboard for everything listening on your machine’s ports. Open the TUI with ports, scan what is running, sort and filter the list, inspect one port, kill a process safely, open localhost in the browser, or jump back into the project folder in your editor.

See Every Listening Port

Port Pilot scans listening TCP ports with lsof, enriches them with PID, memory, uptime, command, project root, and framework, and brings it together in one place.

Act Without Leaving Terminal

Kill a process with confirmation, open localhost in your browser, or jump straight into the project folder in Cursor or VS Code.

Work The Way You Think

Sort by port, memory, uptime, or project. Filter to dev ports or node-based apps. Search live across ports, project names, frameworks, commands, and paths.

Quick Start

First-time setup can be done in one command:

npm run setup

That script runs install, build, and link for you. Manual setup is also available:

npm install
npm run build
npm link

After linking, run ports from anywhere in your terminal.

CLI Commands

ports
Open the live terminal dashboard.
ports list
Print a static table of listening ports.
ports check <port>
Show detailed information for one port.
ports kill <port>
Stop the process listening on a port.
ports open <port>
Open http://localhost:PORT in the default browser.
ports edit <port>
Open the project root in Cursor or VS Code.

How It Works

The CLI is built with TypeScript, Commander.js, Ink, React, chalk, execa, and tsup. Port Pilot uses lsof to discover listening TCP processes, ps to pull memory and uptime, and walks upward from the process working directory to find package.json and detect the framework.

Framework detection currently includes Next.js, Astro, Vite, Remix, Nuxt, SvelteKit, and Angular.

Port Pilot is designed as a daily-use terminal companion for developers who constantly bounce between localhost apps, background servers, and editor windows.
